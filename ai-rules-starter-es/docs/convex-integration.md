# Integración con Convex

Este starter no incluye tooling de Convex.

Cuando un repo destino use Convex, exige evidencia antes de cambios:

- Qué deployment está activo.
- Qué schema, functions y tests son relevantes.
- Si las lecturas o escrituras necesitan credenciales reales.
- Qué cleanup se requiere después de la validación.
